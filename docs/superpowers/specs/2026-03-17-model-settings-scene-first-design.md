# 模型设置 Scene-First 设计

## 背景

当前 `ProviderSettingsView` 把“默认能力 / 自定义服务商 / 当前生效路由”混在同一个交互里，用户需要先理解 provider 再去配置场景，信息架构与用户心智不一致。

用户确认后的目标交互：

- 用户首先只关心“每个场景用哪个模型”
- 添加自定义服务商只是为了让更多模型出现在可选列表
- 模型展示必须带服务商前缀
- 内置默认来源命名为 `内置免费`
- `内置免费` 如果当前未探测到模型，则不进入选择列表
- 添加自定义服务商不会自动切换当前场景模型

## 目标结构

`模型设置` 页面只保留两层：

1. 场景模型分配
2. 从场景选择器进入“添加自定义服务商”次级面板

主界面不再出现“来源区域”“当前生效”“默认能力 vs 自定义服务二选一”。

## 主界面

页面包含三个场景选择器：

- 主对话
- 标题与摘要
- 失败兜底

每个选择器中的可选项统一展示为：

- `内置免费 / <model>`
- `<自定义服务商名称> / <model>`
- `+ 添加自定义服务商`

交互规则：

- 点击 `+ 添加自定义服务商` 打开次级面板
- 添加成功后返回主界面
- 当前场景原有选择保持不变
- 新服务商的模型仅加入可选列表，不自动替用户改场景

## 次级面板

只负责创建或更新一个自定义 OpenAI-compatible 服务商：

- 服务商名称（必填）
- API Base（必填）
- API Key（必填）
- 连接并获取模型

获取成功后，这个服务商的所有模型加入主界面的场景选择器。

## 数据映射

底层仍沿用 `PanelConfigNew`：

- `llmProviders[]`
- `llmProfiles[]`
- `llmDefaultProfile`
- `llmAuxProfile`
- `llmFallbackProfile`

但 UI 不再暴露 profile/provider 概念。

映射原则：

- 每个场景最终都落到一个具体 profile
- 内置免费模型也作为正常模型选项参与映射
- 自定义服务商模型也作为正常模型选项参与映射
- 允许多个自定义服务商并存

## 内置免费模型来源

内置免费模型列表来自现有 Cursor Help inspect 能力：

- content script 已能采集 `selectedModel` / `availableModels`
- service worker 已能读取这组 inspect 数据

本次需要补的是：把这组运行时模型数据接到 panel 的模型目录聚合层。

## 测试要求

- 挂载模型设置页面时不得修改全局配置
- 内置免费模型与自定义服务商模型能被统一聚合
- 选择场景模型后能正确写回 `llmDefaultProfile / llmAuxProfile / llmFallbackProfile`
- 选择 `+ 添加自定义服务商` 只打开次级面板，不改变当前场景值
- 添加服务商成功后，模型目录新增该服务商前缀选项
